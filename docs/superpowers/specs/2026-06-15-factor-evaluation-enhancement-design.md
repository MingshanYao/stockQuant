## 1. 动机与目标

### 1.1 问题

在 Alpha191 因子复现的回测中，观察到以下现象：

- **策略净值曲线1 因子复现的回测中，观察到以下现象：\n\n- **策略净值曲线呈现"漫长亏损 + 剧烈拉升"的锯齿形态** — 策略长时间不赚钱甚至亏钱，但在少数时间段内斜率非常陡峭\n- **现有评价体系无法区分「市场 Beta 驱动」和「选股 Alpha」** — 策略的收益可能主要来自对市场风格的暴露，而非真正的选股能力\n- **交易层面的统计缺失** — 没有胜率、盈亏比、Profit Factor 等核心交易指标\n\n### 1.2 核心目标\n\n参照国泰君安(2017)《基于短周期价量特征的多因子选股体系》中的方法论：\n\n> **股票收益率 = 风格收益 + 阿尔法收益**\n> 评价阿尔法模型的标准应是计算其对阿尔法收益的预测是否可靠\n> 预测与组合实现的目标必须一致\n\n具体目标：\n\n| # | 目标 | 对应需求 |\n|---|------|----------|\n| 1 | 风格中性化的因子 IC 计算 | 剔除市场影响评估因子 |\n| 2 | 组合构建时的行业/市值中性约束 | 确保预测与组合目标一致 |\n| 3 | 超额收益曲线（策略-基准） | 剔除市场影响 #1 |\n| 4 | 剥离 Beta 后的纯 Alpha 曲线 | 剔除市场影响 #2 |\n| 5 | 逐笔交易的 Alpha 盈亏 | 剔除市场影响 #4 |\n| 6 | 交易胜率统计（胜率/盈亏比/Profit Factor/最大连败等） | 交易胜率需求 |\n\n---\n\n## 2. 整体架构\n\n```\n                                  ┌─────────────────────────┐\n                                  │    FactorEvaluator      │ ← 扩展风格中性化\n                                  │   (evaluator.py)        │    行业/市值/Beta/波动/动量\n                                  └──────────┬──────────────┘\n                                             │ 评价结果\n                                             ▼\n                                  ┌─────────────────────────┐\n                                  │  AlphaFactorStrategy     │ ← 优化器约束：行业±5%, 市值±0.1σ\n                                  │   (alpha_factor_strategy)│    \n                                  └──────────┬──────────────┘\n                                             │ 回测结果\n                                             ▼\n                                  ┌─────────────────────────┐\n                                  │  PerformanceAnalyzer     │ ← 新增胜率统计 + Alpha收益\n                                  │   (performance.py)       │    逐笔Alpha盈亏\n                                  └──────────┬──────────────┘\n                                             │ 绩效数据\n                                             ▼\n                                  ┌─────────────────────────┐\n                                  │     PlotEngine           │ ← 三面板图：净值/超额/Alpha\n                                  │   (plot.py)              │    交易仪表盘\n                                  └─────────────────────────┘\n```\n\n**架构原则：** 各模块职责清晰——FactorEvaluator 负责评价、Strategy 负责执行、PerformanceAnalyzer 负责分析、PlotEngine 负责可视化。每个模块的增强保持独立，通过现有的接口（FactorEvaluator -> DataFrame, BacktestResult -> PerformanceAnalyzer -> equity/trade 数据）连接。\n\n---\n\n## 3. FactorEvaluator — 风格中性化 IC 计算\n\n### 3.1 当前状态\n\n- `FactorEvaluator.__init__` 接受 `close_panel`, `industry`, `market_cap`\n- `_neutralize_panel()` 仅支持行业和市值两因子做截面回归去残差\n- `neutralize` 参数默认 `True`，但可为 `False`\n\n### 3.2 改造方案\n\n#### 3.2.1 风格因子数据源扩展\n\n新增参数 `style_factor_data: dict[str, pd.DataFrame]`，键为风格因子名称（`beta`, `volatility`, `momentum`, `turnover`），值为 DataFrame（行=日期，列=股票代码），存储每只股票每日的风格因子暴露值。\n\n若用户未提供，系统自动从 `close_panel` 计算：\n\n| 风格因子 | 计算方式 | 窗口 |\n|----------|----------|------|\n| Beta | 滚动回归股票收益 vs 基准收益 | 60 日 |\n| Volatility | 滚动标准差 | 20 日 |\n| Momentum | 过去收益率 | 20 日 |\n| Turnover | 日换手率（需外部传入） | — |\n\n#### 3.2.2 截面回归中性化（增强版）\n\n```python\ndef _neutralize_panel(self, factor: pd.DataFrame) -> pd.DataFrame:\n    \"\"\"对每日截面做多元线性回归，取残差作为纯Alpha因子值。\n    \n    回归模型：\n        factor_value = β_0 + β_ind·X_ind + β_size·X_size + β_beta·X_beta \n                      + β_vol·X_vol + β_mom·X_mom + ε\n    \n    返回：ε 残差截面\n    \"\"\"\n```\n\n#### 3.2.3 evaluate() 流程\n\n```\n输入: factor_panel, forward_period=1\n  │\n  ├─ 1. 风格中性化因子值（回归取残差） → 纯Alpha预测值 E{ε}\n  │\n  ├─ 2. 对前向收益做同样的风格中性化 → 实际Alpha收益 α\n  │\n  ├─ 3. IC = corr(E{ε_t+1}, α_t+1)   # Rank IC (Spearman)\n  │\n  ├─ 4. Factor Return: 截面回归 R = β_factor × E{ε} + ε'\n  │\n  └─ 5. T统计量: IC_mean / (IC_std / √N)\n```\n\n关键变化：**前向收益也做风格中性化**，这是 alpha191 方法论的核心——只有实际 Alpha 收益才应该用于评价 Alpha 预测能力。\n\n#### 3.2.4 接口变更\n\n```python\n# 新的初始化接口\nclass FactorEvaluator:\n    def __init__(\n        self,\n        close_panel: pd.DataFrame,\n        industry: pd.Series | None = None,        # code → industry\n        market_cap: pd.Series | pd.DataFrame | None = None,  # code→cap, 或 dates×codes panel\n        style_factors: dict[str, pd.DataFrame] | None = None,  # {name: dates×codes panel}\n        benchmark_returns: pd.Series | None = None,  # 用于计算Beta\n    ):\n```\n\n`neutralize` 参数从 `evaluate()` 签名中移除（始终强制中性化）。\n\n---\n\n## 4. AlphaFactorStrategy — 组合构建风格中性化\n\n### 4.1 当前状态\n\n当前策略在每个调仓日：\n\n```python\n# 简单取 Top-N 等权买入\nselected = factor_sorted[:self.max_positions]  # 没有行业/市值约束\n```\n\n### 4.2 改造方案\n\n使用 **线性规划优化器 (cvxpy)** 构建组合：\n\n#### 4.2.1 约束条件\n\n| 约束 | 定义 |\n|------|------|\n| 行业中性 | 组合中各行业的权重与基准的行业权重偏差 ≤ ±5%（相对比例） |\n| 市值中性 | 组合的市值Z-score与基准的市值Z-score偏差 ≤ ±0.1 |\n| 满仓投资 | 总权重 = 1（或 max_positions 只） |\n| 个股上限 | 单只股票权重 ≤ 2%（50只等权时的上限） |\n| 个股下限 | 单只股票 ≥ 0（不允许卖空） |\n\n#### 4.2.2 目标函数\n\n最大化组合的因子值加权分数：\n\n```\nmaximize: Σ w_i × factor_score_i\n```\n\n其中 `factor_score_i` 是每只股票的因子值（已经风格中性化）。\n\n#### 4.2.3 实现方式\n\n```python\nclass AlphaFactorStrategy(BaseStrategy):\n    def __init__(\n        self,\n        ...\n        enable_style_neutral: bool = True,        # 默认开启\n        industry_exposure_limit: float = 0.05,    # ±5%\n        size_exposure_limit: float = 0.1,         # ±0.1σ\n    ):\n        self.enable_style_neutral = enable_style_neutral\n        ...\n\n    def _build_optimized_portfolio(self, factor_panel: pd.DataFrame) -> list[str]:\n        \"\"\"使用 cvxpy 构建行业/市值中性的优化组合。\"\"\"\n```\n\n#### 4.2.4 依赖项\n\n新增依赖：`cvxpy`（线性规划求解器）\n\n```bash\npip install cvxpy\n```\n\n#### 4.2.5 备用方案\n\n若 cvxpy 安装失败或用户选择不使用，提供 `_build_skip_neutral` 作为回退（维持当前行为）。\n\n---\n\n## 5. PerformanceAnalyzer — 交易胜率 + Alpha 收益\n\n### 5.1 交易胜率统计\n\n当前 `trade_statistics()` 仅返回：总交易次数、买入次数、卖出次数、总佣金。\n\n扩展为：\n\n| 字段 | 类型 | 计算方式 |\n|------|------|----------|\n| 总交易次数 | int | len(trades) |\n| 盈利交易次数 | int | sum(PnL > 0) |\n| 亏损交易次数 | int | sum(PnL < 0) |\n| **胜率** | float | 盈利交易数 / 总交易数 |\n| **平均盈利** | float | 盈利PnL总和 / 盈利交易数 |\n| **平均亏损** | float | 亏损PnL总和 / 亏损交易数 |\n| **盈亏比** | float | 平均盈利 / 平均亏损 |\n| **Profit Factor** | float | 盈利PnL总和 / 亏损PnL总和 |\n| 最大连胜 | int | 连续盈利交易的最长序列 |\n| 最大连败 | int | 连续亏损交易的最长序列 |\n| 平均持仓天数（盈利） | float | 盈利交易的平均持仓 |\n| 平均持仓天数（亏损） | float | 亏损交易的平均持仓 |\n| 总佣金 | float | 保持现有 |\n\n计算方式：\n- 交易 PnL 从 `trade_log` 获取（`commission`、`price`、`quantity` 字段）\n- 每笔交易的 PnL = 卖出金额 - 买入金额 - 佣金\n- 持仓天数 = 卖出日期 - 买入日期\n- 连续盈亏通过顺序扫描交易日志计算\n\n### 5.2 逐笔 Alpha 盈亏\n\n```python\ndef per_trade_alpha_pnl(self, benchmark: pd.Series) -> pd.DataFrame:\n    \"\"\"计算每笔交易的 Alpha 盈亏。\n    \n    对每笔交易：\n    1. 计算持仓期间的市场收益率 r_market\n    2. 计算该股票的 Beta（60日滚动回归）\n    3. Alpha PnL = 实际PnL - (Beta × r_market × 投入资金)\n    \n    返回：trade_log + alpha_pnl 列\n    \"\"\"\n```\n\n### 5.3 超额收益与 Alpha 收益序列\n\n```python\ndef excess_returns(self) -> pd.Series:\n    \"\"\"策略日收益率 - 基准日收益率\"\"\"\n\ndef alpha_returns(self, window: int = 60) -> pd.Series:\n    \"\"\"策略日收益率 - (滚动Beta × 基准日收益率)\n    \n    Beta = cov(R_strategy, R_benchmark) / var(R_benchmark)\n    使用滚动窗口计算时变 Beta。\n    \"\"\"\n\ndef cumulative_alpha(self) -> pd.Series:\n    \"\"\"累计Alpha收益曲线（从1.0开始）\"\"\"\n\n```\n\n---\n\n## 6.
